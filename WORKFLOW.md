# WORKFLOW.md - 智简 AI 项目多 Agent 协作流程

> 这是项目的协作流程宪章。任何 Agent 在执行任务前必读此文档。

---

## 核心理念

> **PM 输入"产品目的",AI 团队输出"可上线产物",中间过程 PM 不参与**。

这套协作流程的目标:
- 让 AI 系统具备"自我评分、自我优化"能力
- 把 PM 从"反复授权"中解放出来
- 让 PM 只在"输入"和"验收"两端介入

---

## 三角色定义

| 角色 | 职责 | 输出 | 平台建议 |
|------|------|------|---------|
| **工程师 Agent** | 实现功能 | 工程师产出报告 | Claude Code(本地) |
| **质检员 Agent** | 严苛评估 | 质检评估报告(评分) | OpenAI Codex 或第二个 Claude Code 会话 |
| **整合者 Agent** | 综合决策 | 整合者决策报告 | Claude.ai 或第三个 Claude Code 会话 |

每个角色都有**独立的 prompt 文件**(在 `prompts/` 目录):
- `prompts/engineer.md`
- `prompts/qa.md`
- `prompts/integrator.md`

启动 Agent 时,把对应 prompt 文件的内容**完整粘贴**到对话开头。

---

## 完整工作流

```
┌────────────────────────────────────────────┐
│ 1. PM 填任务模板                          │
│ tasks/0XX-task-name.md                    │
│ (5-10 分钟)                                │
└──────────────────┬─────────────────────────┘
                   ↓
┌────────────────────────────────────────────┐
│ 2. 工程师 Agent 接收任务                   │
│ - 读 CLAUDE.md "已知陷阱清单"              │
│ - 给 PM 提供 A/B/C 方案                    │
└──────────────────┬─────────────────────────┘
                   ↓
┌────────────────────────────────────────────┐
│ 3. PM 选方案(5 分钟)                       │
│ 直接回复"选 A"/"选 B"/"选 C"               │
└──────────────────┬─────────────────────────┘
                   ↓
┌────────────────────────────────────────────┐
│ 4. 工程师 Agent 实现                       │
│ - 写代码/改文件                             │
│ - 走 5 层自检流程                           │
│ - 输出"工程师产出报告"                      │
└──────────────────┬─────────────────────────┘
                   ↓
┌────────────────────────────────────────────┐
│ 5. PM 复制工程师报告 → 质检员 Agent        │
│ (10 秒,纯复制粘贴)                         │
└──────────────────┬─────────────────────────┘
                   ↓
┌────────────────────────────────────────────┐
│ 6. 质检员 Agent 评估                       │
│ - 读任务文档 + 工程师报告 + CLAUDE.md      │
│ - 严苛打分(基础 40 / 体验 30 / 姿态 20 / 长期 10) │
│ - 输出"质检评估报告"                        │
└──────────────────┬─────────────────────────┘
                   ↓
┌────────────────────────────────────────────┐
│ 7. PM 复制工程师 + 质检员报告 → 整合者     │
│ (10 秒)                                    │
└──────────────────┬─────────────────────────┘
                   ↓
┌────────────────────────────────────────────┐
│ 8. 整合者 Agent 决策                       │
│ - 综合两方意见                              │
│ - 决策:通过/返工/重做                       │
│ - 如果通过:给 PM 5-10 分钟测试清单         │
│ - 如果返工:给工程师明确指令                 │
└──────────────────┬─────────────────────────┘
                   ↓
        ┌──────────┴──────────┐
        ↓                     ↓
   返工/重做              通过
        ↓                     ↓
   回到 Step 4          PM 5-10 分钟验收
                              ↓
                         上线/部署
                              ↓
                     PM 沉淀经验到文档
```

---

## PM 介入节点(精确清单)

整个流程,PM 只在以下节点介入:

| 节点 | 时间 | 做什么 |
|------|------|--------|
| 1. 输入任务 | 5-10 分钟 | 填 task 模板 |
| 2. 选方案 | 5 分钟 | 选 A/B/C |
| 3. 复制流转 | 30 秒 × 2 次 | 工程师 → 质检员 → 整合者 |
| 4. 真机验收 | 5-10 分钟 | 按整合者给的清单测试 |

**总投入**:每个任务 ≤ 30 分钟。

## Agent 之间的"信息传递"

### 简单方式(手动复制)

PM 把上一个 Agent 的输出**完整复制**到下一个 Agent 的对话里。

模板:
```
[粘贴工程师产出报告完整内容]

---

请按 prompts/qa.md 的流程评估这个产出。
```

### 进阶方式(文件中转)

工程师 Agent 把报告写入 `tasks/0XX/engineer-report.md`,
质检员 Agent 读这个文件 → 写 `tasks/0XX/qa-report.md`,
整合者 Agent 读这两个 → 写 `tasks/0XX/integrator-report.md`。

每个任务变成一个文件夹:
```
tasks/008-feedback-mechanism/
├── task.md              ← PM 填的任务模板
├── engineer-report.md   ← 工程师输出
├── qa-report.md         ← 质检员输出
├── integrator-report.md ← 整合者决策
└── pm-acceptance.md     ← PM 验收记录
```

**推荐用进阶方式**——所有协作过程留档,未来能复盘和优化 prompts。

## 启动一个新任务的步骤

### Step 1:创建任务

```bash
# 从模板复制
cp tasks/templates/task-template.md tasks/0XX-task-name.md

# 编辑填写 8 个字段
```

### Step 2:启动工程师 Agent

打开 Claude Code(或新会话),粘贴:

```
[prompts/engineer.md 完整内容]

---

我有一个新任务要做:tasks/0XX-task-name.md

请按工程师 Agent 流程开始:先读项目和任务,
然后给我 A/B/C 方案。
```

### Step 3:选方案后实现

工程师产出方案后,你回复:"选 A"(或 B/C)。

工程师开始实现 + 自检 + 输出报告。

### Step 4:启动质检员 Agent

打开新对话(或换平台),粘贴:

```
[prompts/qa.md 完整内容]

---

工程师 Agent 提交了产出报告,请你严苛评估:

[粘贴工程师产出报告完整内容]

任务文档在 tasks/0XX-task-name.md。

请按 QA Agent 流程给出评分和建议。
```

### Step 5:启动整合者 Agent

打开新对话,粘贴:

```
[prompts/integrator.md 完整内容]

---

收到工程师产出和质检员评估,请你做最终决策。

工程师产出报告:
[粘贴]

质检员评估报告:
[粘贴]

请按整合者 Agent 流程输出决策。
```

### Step 6:执行整合者的决定

- 如果"返工":回到 Step 3,把整合者的指令交给工程师
- 如果"通过":按整合者给的"5-10 分钟测试清单"做真机验收

### Step 7:PM 真机验收

按整合者给的清单测试。

如果通过:
- 工程师 Agent 部署/发版
- 整合者 Agent 把新陷阱沉淀到 CLAUDE.md
- 写复盘进 DECISIONS.md

如果不通过:
- 反馈具体问题给工程师 Agent
- 进入返工流程
- 把"质检员没发现 PM 发现的问题"加入 prompts/qa.md(让质检员下次更严)

## 质量门槛

### 90 分门槛

质检员评估总分必须 ≥ 90 才能通过。

低于 90 默认返工,除非:
- PM 明确豁免某些扣分(在整合者建议下)
- 扣分原因是"AI 能力边界",且 PM 接受现状

### 5 层自检强制

工程师 Agent 不走完 5 层自检,质检员直接打回。

### 已知陷阱必读

任何 Agent 开始前必须读 CLAUDE.md "已知陷阱清单"。

不读 = 失职。

---

## 异常处理

### 异常 1:返工 2 次还没解决同一问题

→ **升级给 PM**。
工程师 Agent 输出"我尝试了 X/Y 方案,都没通过质检员的 ABC 评估,需要 PM 介入"。

### 异常 2:工程师和质检员意见严重冲突

工程师说"我做对了"、质检员说"完全不对"——
→ **整合者升级给 PM**,带上双方完整论据。

### 异常 3:发现新的"AI 能力边界"

工程师明确说"做不到"且质检员同意——
→ **整合者标注"AI 能力边界",升级给 PM**。
→ PM 决定:补能力 / 接受 / 改方案。

### 异常 4:任务超出预期工作量

如果实际工作量 > 估计 2 倍——
→ **暂停,升级给 PM 重新评估**。

## 流程进化

### 每次任务后的复盘

完成一个任务后,Agent 团队主动复盘:
- 哪个环节顺?
- 哪个环节卡?
- prompts 哪里不够好?
- 是否需要新增"已知陷阱"?

### Prompts 迭代

prompts 文件**不是静态的**,会持续优化:
- 每周复盘一次
- 把发现的协作问题转化为 prompts 的明确条款
- 让流程一周比一周顺

### 已知陷阱清单膨胀

每次新发现的坑都进 CLAUDE.md "已知陷阱清单"。
当清单太长时(超过 30 条),做一次"陷阱重构":
- 合并相似条目
- 移除已经不再适用的
- 按主题分组

---

## 重要边界

### 这套流程**不是**

- ❌ 完全自动化(PM 还是要介入 4 个节点)
- ❌ 替代 PM 的产品判断
- ❌ 替代 PM 的真机验收
- ❌ 适用于战略决策(只用于工程任务)

### 这套流程**是**

- ✅ 让 AI 团队自驱动 80% 的工程过程
- ✅ 让 PM 投入 30 分钟搞定原本要 8 小时的任务
- ✅ 让 AI 之间互相挑战,找出彼此盲区
- ✅ 让经验自动沉淀,同类问题不再犯

## 下一阶段路线图

### v1(本周完成):手动协作流
- 三角色 prompts
- 手动复制粘贴流转
- 文件中转记录

- 写脚本自动把 Agent A 的输出传给 Agent B
- 用 GitHub Actions 触发整个流程
- PM 只填 task.md,其他自动跑

### v3(1 个月):数据驱动
- 启动器内嵌反馈
- 用户数据自动收集
- AI 根据数据决定下一个修什么

### v4(3 个月):真正的自我进化
- 参考 EvoAgentX / SICA
- Agent 自己优化自己的 prompts
- 长期飞轮形成
